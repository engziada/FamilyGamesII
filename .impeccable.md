## Design Context

### Users
Arabic-speaking teenagers and adults at parties and social gatherings. They're using the platform during game nights, celebrations, and friend meetups. The job-to-be-done is entertainment and social bonding - creating memorable moments through friendly competition. Users may have varying tech proficiency and may be playing in noisy, social environments.

### Brand Personality
**Playful, Warm, Inclusive.** The tone is celebratory and fun - like a family gathering where everyone's invited. The design should feel like a party: energetic but not overwhelming. Think of it as the digital equivalent of a beautifully set dinner table where the food is the games.

### Aesthetic Direction
- **Visual tone:** Vibrant, energetic, slightly whimsical - asymmetric borders, playful animations,emoji reactions
- **Theme:** Dual mode - bright/cheerful light theme, cozy/moody dark theme
- **References:** The playful asymmetry recalls the joy of handmade crafts; the bright colors feel like festival decorations
- **Anti-patterns:** Avoid sterile corporate aesthetics, excessive minimalism, or muted/pastel palettes that drain energy

### Design Principles

1. **Joy First** - Every interaction should spark delight. Use animations, colors, and micro-interactions that make people smile. The current heartbeat, jelly, and pulse animations embody this.

2. **Cognitive Ease** - Reduce mental load during gameplay. Clear visual hierarchy, obvious affordances, large touch targets (44px+), and intuitive flows. Players shouldn't need to think about how to use the interface.

3. **RTL-Native** - Arabic is not an afterthought. Every component is designed right-to-left first, with proper mirroring and culturally appropriate spacing.

4. **Inclusive Celebration** - Confetti, score animations, and emoji reactions celebrate everyone's achievements. Design for shared moments, not individual competition.

5. **Bold & Playful** - Embrace the current asymmetric border-radius (`15px50px 15px 50px`), pop shadows (`5px5px`), and vibrant colors (`#FF6B6B`, `#4ECDC4`, `#FFE66D`). The current design direction is correct - enhance, don't tone down.

### Typography
- **Headings:** Lemonada (cursive, Arabic-friendly) - weights 300-700
- **Body:** Changa (sans-serif, Arabic-friendly) - weights 300-800
- **Scale:** Mobile-first with `clamp()` for fluid sizing

### Spacing & Motion
- **Shadows:** Pop (`5px 5px`), Heavy (`8px 8px`) - playful depth
- **Border-radius:** Asymmetric (`30px 80px30px 70px / 70px 30px 80px 30px`) - whimsical personality
- **Transitions:** `0.4s cubic-bezier(0.175, 0.885, 0.32, 1.275)` - bouncy, playful feel
- **Animations:** Heartbeat, jelly, bounceInDown, pulse - match current sprinkle-pattern background

### Accessibility Notes
- Skip links implemented ✓
- Focus visible outlines ✓
- Minimum 44px touch targets ✓
- System dark mode detection ✓
- **Enhance for cognitive ease:** Clear visual hierarchy, avoid clutter, provide clear feedback states

## Design System

### File Structure

```
static/
├── css/
│   ├── tokens.css          # Design tokens (colors, spacing, typography)
│   └── style.css           # Main styles (uses tokens)

        ├── index.js         # Component library entry point
        ├── button.js        # Button component
        ├── card.js          # Card component
        ├── modal.js         # Modal component
        ├── input.js         # Input component
        ├── badge.js         # Badge component
        ├── toast.js         # Toast notification component
        └── player-item.js   # Player list item component
```

### Design Tokens (`tokens.css`)

Core design tokens are defined as CSS custom properties:

#### Color Tokens
```css
--color-primary: #FF6B6B;      /* Coral - CTAs, highlights */
--color-secondary: #4ECDC4;    /* Teal - Secondary actions */
--color-accent: #FFE66D;        /* Yellow - Warnings, emphasis */
--color-success: #6BCB77;       /* Green - Correct answers */
--color-danger: #FF6B6B;        /* Coral - Errors, warnings */
--color-purple: #BC7AF9;        /* Purple - Team distinctions */
```

#### Spacing Tokens
```css
--space-1: 0.25rem;   /* 4px */
--space-2: 0.5rem;    /* 8px */
--space-3: 0.75rem;   /* 12px */
--space-4: 1rem;      /* 16px */
--space-5: 1.5rem;    /* 24px */
--space-6: 2rem;      /* 32px */
--space-8: 3rem;      /* 48px */
```

#### Border Radius Tokens (Signature Asymmetric Style)
```css
--radius-card: 30px 80px 30px 70px / 70px 30px 80px 30px;
--radius-input: 15px 30px 10px 25px;
--radius-button: 15px 50px 15px 50px;
```

### Components

#### Button (`button.js`)
Creates consistent buttons with variants and sizes.

```javascript
// Primary button
Button.create({ text: 'بدء اللعبة', variant: 'primary', icon: 'fa-play' });

// Icon-only button
Button.create({ icon: 'fa-cog', variant: 'outline', ariaLabel: 'Settings' });

// Button variants: primary, secondary, success, danger, outline
// Button sizes: sm, md, lg
```

#### Card (`card.js`)
Cards with the signature asymmetric border-radius.

```javascript
// Basic card
Card.create({ title: 'Game Title', body: 'Description' });

// Game card for catalog
Card.createGame({
  title: 'بدون كلام',
  icon: 'fa-mask',
  players: '2-8 لاعبين',
  onPlay: () => { /* start game */ }
});

// Player list card
Card.createPlayerList('اللاعبين');

// Scoreboard card
Card.createScoreboard('النتائج');
```

#### Modal (`modal.js`)
Modal dialogs with consistent styling.

```javascript
// Basic modal
Modal.create({
  id: 'my-modal',
  title: 'تأكيد',
  body: '<p>هل أنت متأكد؟</p>',
  buttons: [
    { text: 'إلغاء', variant: 'secondary', dismiss: true },
    { text: 'تأكيد', variant: 'primary', onClick: () => {} }
  ]
});

// Quick helpers
Modal.confirm({ message: 'هل أنت متأكد؟', onConfirm: () => {} });
Modal.alert({ message: 'تم بنجاح!', icon: 'fa-check-circle' });
```

#### Input (`input.js`)
Form inputs with RTL support.

```javascript
// Text input
Input.text({ id: 'player-name', label: 'اسمك', placeholder: 'اكتب اسمك هنا...' });

// Select dropdown
Input.select({
  id: 'game-type',
  label: 'نوع اللعبة',
  options: [{ value: 'charades', label: 'بدون كلام' }]
});

// Game input (for bus complete, etc.)
Input.createGameInput({ placeholder: 'الكلمة...' });
```

#### Badge (`badge.js`)
Status badges, team badges, and score displays.

```javascript
// Status badge
Badge.create({ text: 'مستخدم جديد', variant: 'info' });

// Team badge
Badge.team(1); // Returns badge for Team 1

// Host badge
Badge.host(); // Returns "مضيف" badge

// Score badge
Badge.score(15); // Returns score pill

// Medal for rankings
Badge.medal(1); // Returns "🥇"
```

#### Toast (`toast.js`)
Toast notifications.

```javascript
// Basic toast
Toast.show('تم النسخ!', 'success');

// Quick helpers
Toast.success('تم بنجاح!');
Toast.error('حدث خطأ');
Toast.warning('تحذير');
Toast.info('معلومة');
```

#### PlayerItem (`player-item.js`)
Player list items for lobbies and scoreboards.

```javascript
// Player list item
PlayerItem.create({
  name: 'أحمد',
  score: 15,
  avatar: '🦁',
  isHost: true,
  isMe: true,
  isConnected: true
});

// Scoreboard row
PlayerItem.createScoreRow({
  name: 'أحمد',
  score: 15,
  rank: 1,
  avatar: '🦁'
});

// Update entire list with animation
PlayerItem.updateList(listEl, players, currentPlayerName);
```

### Usage

Import components in your JavaScript:

```javascript
// Import individual components
import { Button } from '/static/js/components/button.js';
import { Card } from '/static/js/components/card.js';
import { Toast } from '/static/js/components/toast.js';

// Or use the global namespace (when loaded via script tag)
const btn = window.FGComponents.Button.create({ text: 'Play' });
```

Load tokens.css before style.css:

```html
<link rel="stylesheet" href="/static/css/tokens.css">
<link rel="stylesheet" href="/static/css/style.css">
```

### Animation Classes

Use these utility classes for animations:

- `.animate-jelly` - Bouncy entrance
- `.animate-heartbeat` - Pulsing attention
- `.animate-bounce-down` - Drop-in animation
- `.fade-in` - Fade in from below
- `.score-fly-up` - Floating "+X" score animation

### Accessibility

All components include:
- Proper ARIA attributes
- Focus management
- RTL support
- Minimum 44px touch targets
- High contrast focus indicators (`--focus-ring`)
