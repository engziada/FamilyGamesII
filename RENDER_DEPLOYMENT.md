# Render.com Deployment Guide

## Prerequisites

1. A [Render.com](https://render.com) account (free tier available)
2. Your GitHub repository connected to Render
3. Convex deployment URL (from your Convex project)
4. Groq API key (if using AI features)

## Deployment Steps

### 1. Push Code to GitHub

Ensure all deployment files are committed:
```bash
git add render.yaml build.sh RENDER_DEPLOYMENT.md
git commit -m "Add Render.com deployment configuration"
git push origin main
```

### 2. Create New Web Service on Render

1. Log in to [Render Dashboard](https://dashboard.render.com/)
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository
4. Select the `FamilyGamesII` repository
5. Render will auto-detect the `render.yaml` configuration

### 3. Configure Environment Variables

In the Render dashboard, add the following environment variables:

| Variable | Value | Notes |
|----------|-------|-------|
| `SECRET_KEY` | Auto-generated | Render will generate this automatically |
| `CONVEX_URL` | `https://your-project.convex.cloud` | Get from Convex dashboard |
| `GROQ_API_KEY` | Your Groq API key | Get from https://console.groq.com/ |

### 4. Deploy

1. Click **"Create Web Service"**
2. Render will automatically:
   - Run `build.sh` to install dependencies
   - Start the app with `gunicorn app:app`
3. Wait for deployment to complete (usually 2-5 minutes)

### 5. Access Your Application

Once deployed, your app will be available at:
```
https://family-games-ii.onrender.com
```

## Important Notes

### Free Tier Limitations

- **Cold starts**: Free tier services spin down after 15 minutes of inactivity
- **First request**: May take 30-60 seconds to wake up
- **Upgrade**: Consider paid tier ($7/month) for always-on service

### Environment Variables

- `SECRET_KEY`: Auto-generated by Render for security
- `CONVEX_URL`: Must match your Convex deployment
- `GROQ_API_KEY`: Required for trivia translation features

### Logs and Monitoring

- View logs: Render Dashboard → Your Service → Logs tab
- Monitor health: Check the service status indicator
- Debug issues: Logs show all Flask output and errors

### Updating the Application

Render auto-deploys on every push to your main branch:
```bash
git add .
git commit -m "Update feature"
git push origin main
```

Render will automatically rebuild and redeploy.

### Manual Deploy

To manually trigger a deploy:
1. Go to Render Dashboard
2. Select your service
3. Click **"Manual Deploy"** → **"Deploy latest commit"**

## Convex Integration

Your app uses Convex for real-time game logic. Ensure:

1. **Convex is deployed**: Run `npx convex deploy` in your local project
2. **CONVEX_URL is set**: Copy from Convex dashboard to Render environment variables
3. **Convex functions are updated**: Any changes to `convex/` folder require redeployment

### Convex Deployment

```bash
# Deploy Convex backend
npx convex deploy

# Get your deployment URL
# Copy the URL from output and add to Render env vars
```

## Troubleshooting

### Build Fails

- Check `build.sh` has execute permissions
- Verify all dependencies in `requirements.txt` are valid
- Check Render build logs for specific errors

### App Won't Start

- Verify `gunicorn` is in `requirements.txt`
- Check that `app.py` has `app` variable defined
- Review startup logs in Render dashboard

### Environment Variables Missing

- Ensure all required env vars are set in Render dashboard
- Restart service after adding new environment variables

### Convex Connection Issues

- Verify `CONVEX_URL` is correct (no trailing slash)
- Check Convex deployment is active
- Ensure Convex functions are deployed
